Step 1. Make sure Git is installed and configured

Open RStudio Tools Global Options Git/SVN

  • Check that Enable version control interface for RStudio projects is checked.

  • If Git isnt found, install it:

    • Windows: Download Git

    • Mac: brew install git

    • Linux: sudo apt install git

Then configure your identity (in the RStudio Terminal or your system terminal):

bash
git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Step 2. Initialize Git in your RStudio project

  1. In RStudio, go to Tools Version Control Project Setup...

  2. Choose Git as the version control system.

  3. RStudio will restart the project and youll now see a Git tab in the top-right pane.

Alternatively, you can do it manually in the Terminal:

bash
git init

Step 3. Create a new GitHub repository

  1. Go to GitHub.

  2. Create a new repository:

    • Name it the same as your project (optional but recommended).

    • Do NOT initialize with a README, .gitignore, or license since RStudio already has a working directory.

Copy the repositorys HTTPS or SSH URL, e.g.:

arduino
https://github.com/yourusername/your-repo.git

Step 4. Connect your local repo to GitHub

In the RStudio Terminal, run:

bash
git remote add origin https://github.com/yourusername/your-repo.git

Check its linked:

bash
git remote -v

Step 5. Add, commit, and push your files

In the Terminal (or using the Git pane in RStudio):

bash
git add .
git commit -m "Initial commit"
git branch -M main
git push -u origin main

Step 6. Verify

Go to your GitHub repo in your browser you should now see your RStudio project files there.

Ah great question, and very smart to ask

Lets break this down simply:

Whats happening

When you opened this new RStudio project and ran Git commands, Git saw that:

  • You were in a folder that already had a Git setup (.git folder inside).

  • That .git folder was still connected to your old GitHub repo (from your previous project).

So even though you thought this was a new project, RStudio was actually still inside or copied from the old Git repository.

Thats why:

  • It said things like
    origin https://github.com/MECH476-FA25/Robert-MECH476.git

  • And your push went to that same GitHub repo.

Why this happens

RStudio projects can sometimes carry over Git info if you:

  • Copied a folder that already had a .git folder inside it, or

  • Opened a subfolder of an existing Git project, or

  • Created a new RStudio project inside another Git-controlled directory.

What you can do now

If you want this really to be a brand new project with its own GitHub repo:

Option 1 Start fresh (recommended)

  1. In Finder/File Explorer, go to your project folder.

  2. Delete the hidden .git folder inside it.
    (On Mac: press Cmd + Shift + . to show hidden files.)

  3. Back in RStudio, go to
    Tools Version Control Project Setup
    and select Git to reinitialize it.

  4. Then connect it to your new GitHub repo using:

    bash
    git remote add origin https://github.com/yourusername/new-repo.git
git push -u origin main

Option 2 Keep it linked to the old repo

If you want this project to stay part of your old GitHub repo (same class repo, for example), you dont need to change anything its already connected and working fine.

TL;DR

  • You didnt do anything wrong.

  • Its normal if your new project lives inside an existing Git repo.

  • You just need to decide:
